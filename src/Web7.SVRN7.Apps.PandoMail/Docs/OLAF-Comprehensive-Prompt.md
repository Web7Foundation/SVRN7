## What This Project Is

**OLAF** (Outlook Look And Feel) is a C# Windows Forms desktop application that recreates the Microsoft Outlook 2003 three-pane email client UI using only standard .NET 2.0 Windows Forms controls. It was written by Microsoft (~2004) as an MSDN sample demonstrating advanced WinForms capabilities.

- **Namespace:** `System.Windows.Forms.Samples`
- **Assembly:** OLAF
- **Framework:** .NET Framework 4.0 (originally .NET 2.0)
- **Platform:** Windows only (Win32, WinForms)
- **No third-party libraries.** All UI is built from `ToolStrip`, `DataGridView`, `TreeView`, `SplitContainer`, `WebBrowser`, and custom owner-drawn painting.

## Data Model

### `MailMessage` (`MailServer/MailMessage.cs`)

Plain data object. Implements `INotifyPropertyChanged` — every property setter calls `OnPropertyChanged`.

| Property | Type | Description |
|---|---|---|
| `From` | `string` | Sender display name |
| `To` | `string` | Recipient display name |
| `Cc` | `string` | CC recipients |
| `Subject` | `string` | Email subject line |
| `Read` | `bool` | Whether the message has been opened |
| `SentDate` | `DateTime` | When the message was sent |
| `Path` | `string` | Filename of the HTML body (e.g., `Mark.htm`) |

### `MessageStore` (`MailServer/MessageStore.cs`)

Singleton. `GetMessageStore()` returns the single instance, constructing it on first call.

**Initialization:** Reads `Mail/Inbox.xml` from the embedded assembly manifest resource stream via `DataSet.ReadXml`. XML schema columns: `From`, `To`, `CC`, `Subject`, `SentDate`, `Read`, `Path`. Populates a `SortableBindingList<MailMessage>` and counts initial unread messages.

**State:**

| Property | Type | Initial | Description |
|---|---|---|---|
| `Messages` | `BindingList<MailMessage>` | from XML | All inbox messages |
| `SelectedMessage` | `MailMessage` | first item | Currently displayed message |
| `UnreadCount` | `int` | counted from XML | Unread message count |
| `DraftsCount` | `int` | 3 (hardcoded) | Drafts folder count |
| `DeletedCount` | `int` | 16 (hardcoded) | Deleted Items count |

**`SelectedMessage` setter side effect:** If the newly selected message is unread, marks it `Read = true` and decrements `UnreadCount`, then fires `PropertyChanged("UnreadCount")`.

**`Reset()`:** Fires `PropertyChanged(null)` to signal all properties changed — used to force full UI refresh.

Implements `INotifyPropertyChanged`. Observers: `FolderView`, `RightSpine`, `MainForm`.

### `SortableBindingList<T>` and `PropertyComparer<T>`

Both live in `MessageStore.cs`.

- `SortableBindingList<T>` extends `BindingList<T>` and overrides `SupportsSortingCore`, `ApplySortCore`, `IsSortedCore`, `RemoveSortCore` to enable column-header sorting on the `DataGridView`.
- `PropertyComparer<T>` is a generic `IComparer<T>` that resolves the named property via `PropertyInfo` reflection and compares via `IComparable` if available, otherwise string comparison. Supports `ListSortDirection.Ascending` / `Descending`.

## Controls

### `Program` (`Program.cs`)

Entry point. `[STAThread]`, enables visual styles, sets compatible text rendering, runs `MainForm`.

---

### `MainForm` (`MainForm.cs`)

Top-level `Form`. Loads at startup:
1. Gets `MessageStore` singleton
2. Sets item count label text with `_store.Messages.Count`
3. Loads online/offline bitmaps from `Properties.Resources.Outlook` and `Properties.Resources.Error`
4. Registers `NetworkChange.NetworkAvailabilityChanged` to call `UpdateStatusBar()` via `this.Invoke`
5. Updates the status bar immediately
6. Sets form icon from `Properties.Resources.Outlook`
7. Registers `SystemEvents.UserPreferenceChanged` to reset `this.Font = SystemFonts.IconTitleFont` and call `PerformAutoScale()`

**`UpdateStatusBar()`:** Checks `NetworkInterface.GetIsNetworkAvailable()`. Online → "All Folders are up to date." + Outlook icon. Offline → "This folder was last updated on {date}." + Error icon.

The designer (`MainForm.Designer.cs`) is large (~1600 lines) and sets up the full three-pane layout with `SplitContainer` instances, `StatusStrip`, and all child controls.

---

### `BaseStackStrip` (`Custom Controls/BaseStackStrip.cs`)

Base class for toolbar strips. Extends `ToolStrip` with `[ToolboxItem(false)]`.

**Constructor defaults:**
- `Dock = DockStyle.Fill`
- `GripStyle = Hidden`
- `Margin = Padding(0)`
- `CanOverflow = false`
- `AutoSize = false`

**Renderer installation:** `SetRenderer()` is called in constructor and in `OnRendererChanged`. It detects if the current renderer is a `ToolStripProfessionalRenderer` and swaps it for a private instance with `RoundedEdges = false` and a `RenderToolStripBackground` override that calls the virtual `OnRenderToolStripBackground`. This pattern works around a WinForms bug where setting the renderer in the constructor doesn't always take effect.

**`OnRenderToolStripBackground` (virtual, overridable):** Default implementation paints a vertical `LinearGradientBrush` using `ToolStripGradientMiddle` → `ToolStripGradientEnd` colors, then draws three border lines (top, left, right) in `SystemPens.ControlDarkDark`.

**`OnSetRenderer(ToolStripProfessionalRenderer)` (virtual):** Hook for subclasses to attach additional render event handlers on the shared renderer instance.

**`OnSetFonts()` (virtual):** Called in constructor and on `UserPreferenceChanged`. Base does nothing; subclasses override to set fonts from system values.

### `StackStrip` (`Custom Controls/StackStrip.cs`)

Extends `BaseStackStrip`. The vertical navigation button bar on the left (Mail, Calendar, Contacts, etc.).

- `LayoutStyle = VerticalStackWithOverflow`
- `OnSetFonts()`: Sets `this.Font` to a bold version of `SystemFonts.IconTitleFont`; fires `ItemHeightChanged` if the height changed
- `OnSetRenderer()`: Hooks `RenderButtonBackground` to paint button backgrounds with gradient fills: pressed/checked → `ButtonPressed*`, hover → `ButtonSelected*`, normal → `ImageMarginGradient*` colors from the color table
- `OnRenderToolStripBackground()`: Empty override — background is transparent (the splitter panel shows through)
- **Radio-group enforcement:** `OnItemAdded` hooks `CheckedChanged` on each added `ToolStripButton`. `StackStripButton_CheckedChanged` uses an `_ignore` flag to prevent recursion while ensuring exactly one button is always checked
- `ItemCount`, `InitialDisplayCount` (min of count and 4), `ItemHeight` (height of first item), `ItemHeightChanged` event

**Seven navigation buttons** (defined in `LeftSpine.Designer.cs`):

| Button | Text | Tag | Image Resource |
|---|---|---|---|
| 1 | Mail | "Read" | `Mail24` |
| 2 | Calendar | "NewAppointment" | `Calendar24` |
| 3 | Contacts | "NewContact" | `Contacts24` |
| 4 | Tasks | "NewTask" | `Tasks24` |
| 5 | Notes | "Notes" | `Notes32` |
| 6 | Folders | "Folder" | (from .resx) |
| 7 | Shortcuts | "Shortcut" | `Shortcut24` |

Button 1 (Mail) starts checked. `ImageTransparentColor = Color.FromArgb(238, 238, 238)` on all buttons.

### `HeaderStrip` (`Custom Controls/HeaderStrip.cs`)

Extends `ToolStrip`. Title bar above content areas.

**`AreaHeaderStyle` enum:** `Large = 0` / `Small = 1`

**`HeaderStyle` property:** Triggers `SetHeaderStyle()`:
- `Large`: Font = `"Arial"` at `MenuFont.SizeInPoints + 3.75F`, bold; `ForeColor = White`; gradient uses `OverflowButtonGradientMiddle → OverflowButtonGradientEnd`
- `Small`: Font = `SystemFonts.MenuFont`; `ForeColor = Black`; gradient uses `MenuStripGradientEnd → MenuStripGradientBegin` (reversed)

Height is auto-calculated from a probe `ToolStripLabel` with text "I" plus 6px padding.

**Usage in LeftSpine:**
- `headerStrip2` (top): Large style, "Mail" label, white text on blue gradient
- `headerStrip1` (below): Small style, "All Mail Folders" label, black text on grey gradient

**Usage in MessageArea (center):** Large style, "Inbox" label.

### `LeftSpine` (`Custom Controls/LeftSpine.cs` + `.Designer.cs`)

`UserControl` decorated with `[Designer(typeof(ParentControlDesigner))]`.

**Structure (from designer):**
```
LeftSpine (BorderStyle.FixedSingle, BackColor=Transparent)
  └── stackStripSplitter (SplitContainer, horizontal, FixedPanel=Panel2)
        ├── Panel1
        │     ├── headerStrip2 (HeaderStrip, Large, "Mail")
        │     ├── headerStrip1 (HeaderStrip, Small, "All Mail Folders")
        │     └── folderView1 (FolderView, Dock=Fill)
        └── Panel2
              ├── stackStrip (StackStrip, Dock=Fill, 7 buttons)
              └── overflowStrip (BaseStackStrip, Dock=Bottom, height=ItemHeight)
                    └── toolStripDropDownButton1 (right-aligned, "Add or Remove Buttons" menu)
```

**On load (`StackBar_Load`):**
1. Hooks `stackStrip.ItemHeightChanged` → `InitializeSplitter()`
2. Calls `AddOverflowItems()` — iterates `stackStrip` buttons in reverse, creates icon-only `ToolStripButton` copies for the overflow strip (using `Tag` to look up bitmap resources)
3. Calls `InitializeSplitter()` — sets splitter distance to show `InitialDisplayCount` (4) buttons
4. Sets `this.Parent.Padding = new Padding(3, 3, 0, 3)`

**Splitter behavior (`stackStripSplitter_SplitterMoved`):**
- If dragged past max height (all buttons visible), clamps to max
- Otherwise recalculates `_visibleCount = stackStrip.Height / stackStrip.ItemHeight`
- Updates overflow item visibility: overflow icons are shown when their corresponding StackStrip button is scrolled out of view

**Splitter painting (`splitContainer1_Paint`):** Paints a gradient bar with up to 9 grip dots (dark/light/mid squares of size 4px with a 2px box) centered horizontally.

**`InitializeSplitter()`:** Sets `SplitterIncrement = ItemHeight`, calculates `SplitterDistance` to show `_visibleCount` buttons, sets `Panel1MinSize`, updates `overflowStrip.Height`.

### `MessageList` (`Custom Controls/MessageList.cs` + `.resx`)

`UserControl` wrapping a `DataGridView` in virtual mode.

**DataGridView configuration:**
- `VirtualMode = true`
- `ReadOnly = true`
- `SelectionMode = FullRowSelect`
- `AllowUserToAddRows/DeleteRows/ResizeRows = false`
- `RowHeadersVisible = false`
- `CellBorderStyle = None`
- `BorderStyle = None`
- `EnableHeadersVisualStyles = false`

**Three columns:**

| Column | Name | Width | DataPropertyName | Notes |
|---|---|---|---|---|
| 0 | SentColumn | 25px | — | `DataGridViewImageColumn`, virtual, read/unread icon |
| 1 | To | Fill | "To" | Custom cell painting (two-line merged cell) |
| 2 | SentDate | 105px | "SentDate" | Date column, header "Newest on top" |

Column 1 header text: "Arranged By: Date". Date format: `"ddd M/dd"` (e.g., "Mon 3/15"); if sent today, switches to short time format `"t"`.

**`CellValueNeeded`:** For column 0, returns `_readImage` (transparent color `RGB(238,238,238)`) or `_unreadImage` based on `MailMessage.Read`.

**`CellPainting` (columns 0 and 1):**
- Column 0: Paints with `DataGridViewPaintParts & ~Focus` to suppress focus rectangle
- Column 1 (merged two-line cell):
  - Fills background with selection or normal background color
  - Line 1: Sender (`From`) in bold if unread, normal if read
  - Line 2: Subject, in grey (`Color.Gray`) if not selected, else selection foreground
  - Both lines clipped with `EndEllipsis`
  - Sets `e.Handled = true`

**`RowPostPaint`:** Selected row gets `DrawFocus`; all others get a `LightGray` border rectangle.

**On load:**
- Binds `messageBS.DataSource = _store.Messages`
- Sorts descending by `SentDate` (column index 2)
- Calls `SetFont()` to set row template height to `(TextHeight + 5) * 2`

**`messageBS_PositionChanged`:** Sets `_store.SelectedMessage = messageBS.Current as MailMessage`.

## Data Flow (End-to-End)

```
Startup
  → MessageStore.GetMessageStore()
      → reads Mail/Inbox.xml (embedded resource)
      → populates SortableBindingList<MailMessage>
      → sets SelectedMessage = Messages[0]

MessageList.Load
  → binds BindingSource to _store.Messages
  → DataGridView.Sort(SentDate, Descending)

User clicks a row
  → BindingSource.PositionChanged
  → MessageStore.SelectedMessage = current MailMessage
      → if !message.Read: message.Read = true; UnreadCount--
      → fires PropertyChanged("SelectedMessage")
      → fires PropertyChanged("UnreadCount")

PropertyChanged("SelectedMessage")
  → RightSpine.MessageStore_PropertyChanged
      → RightSpine.Message = _store.SelectedMessage
          → updates header labels
          → loads Mail/*.htm via webBrowser1.DocumentStream

PropertyChanged("UnreadCount")
  → FolderView.MessageStore_PropertyChanged
      → FolderView.UnreadCount = _store.UnreadCount
          → folderTreeView.Invalidate()
              → folderTreeView_DrawNode
                  → DrawAnnotatedText("Inbox", count, green, "[", "]")
```

## Rendering Techniques

### Gradient Painting
Used everywhere via `LinearGradientBrush`. Always guard with:
```csharp
if ((bounds.Width > 0) && (bounds.Height > 0))
{
    using (Brush b = new LinearGradientBrush(bounds, start, end, mode))
        g.FillRectangle(b, bounds);
}
```

### Professional Color Table
Colors come from `ToolStripProfessionalRenderer.ColorTable` (a `ProfessionalColorTable`). Key color uses:
- Strip background: `ToolStripGradientMiddle` → `ToolStripGradientEnd`
- HeaderStrip large: `OverflowButtonGradientMiddle` → `OverflowButtonGradientEnd`
- HeaderStrip small: `MenuStripGradientEnd` → `MenuStripGradientBegin` (reversed)
- Button normal: `ImageMarginGradientMiddle` → `ImageMarginGradientEnd`
- Button pressed/checked: `ButtonPressedGradientBegin` → `ButtonPressedGradientEnd`
- Button hover: `ButtonSelectedGradientBegin` → `ButtonSelectedGradientEnd`
- Splitter grip dots: `GripDark`, `ToolStripBorder`, `ToolStripDropDownBackground`

### Owner-Drawn DataGridView (MessageList)
The two-line merged cell in column 1 is entirely custom-painted in `CellPainting`. The key pattern:
```csharp
// Draw first line (sender, bold if unread)
TextRenderer.DrawText(g, from, font, rect1, fcolor, TextFormatFlags.EndEllipsis | TextFormatFlags.PreserveGraphicsClipping);
// Draw second line (subject, grey if not selected)
TextRenderer.DrawText(g, subject, normalFont, rect2, greyOrSelected, TextFormatFlags.EndEllipsis | TextFormatFlags.PreserveGraphicsClipping);
e.Handled = true; // suppress default painting
```

### System Font Awareness
Every control registers `Microsoft.Win32.SystemEvents.UserPreferenceChanged` and resets its fonts. The chain:
1. `MainForm` → resets `this.Font = SystemFonts.IconTitleFont`, calls `PerformAutoScale()`
2. `BaseStackStrip` → calls `OnSetFonts()` (virtual)
3. `StackStrip.OnSetFonts()` → bold `IconTitleFont`, fires `ItemHeightChanged`
4. `LeftSpine.InitializeSplitter()` → recalculates splitter from new `ItemHeight`
5. `HeaderStrip` → recalculates header height from new `MenuFont`
6. `MessageList` → calls `SetFont()`, resets `RowTemplate.Height`
7. `FolderView` → calls `SetFont()`, resets node fonts

## Key Design Patterns

### Singleton MessageStore
```csharp
// Correct access — never construct directly
MessageStore store = MessageStore.GetMessageStore();
```

### INotifyPropertyChanged as the Event Bus
`MessageStore` is the central data hub. Controls bind to or observe it:
- `MessageList` uses `BindingSource` data binding (automatic updates)
- `FolderView` and `RightSpine` listen to `PropertyChanged` manually

### Renderer Bug Workaround
Setting a custom renderer in `ToolStrip` constructor can fail because `OnRendererChanged` fires before the constructor completes. The fix in `BaseStackStrip`:
```csharp
// Called from both constructor AND OnRendererChanged
private void SetRenderer()
{
    if ((this.Renderer is ToolStripProfessionalRenderer) && (this.Renderer != _pr))
    {
        if (_pr == null)
        {
            _pr = new ToolStripProfessionalRenderer();
            _pr.RoundedEdges = false;
            _pr.RenderToolStripBackground += ...;
        }
        this.Renderer = _pr;
    }
}
```

### Radio-Group ToolStripButtons
`StackStrip` enforces exactly-one-checked by hooking `CheckedChanged` with an `_ignore` reentrancy guard. When unchecking the last checked button, it re-checks it (preventing an empty selection).

### SplitContainer Snapping
`stackStripSplitter.SplitterIncrement = stackStrip.ItemHeight` makes the splitter snap to button-height grid lines. Clamping to max is done manually in `stackStripSplitter_SplitterMoved`.

## Gotchas and Constraints

1. **`MessageArea` ≠ reading pane.** `MessageArea` is the center list column (contains `MessageList`). `RightSpine` is the reading pane with the `WebBrowser`.

2. **Hardcoded Drafts/Deleted counts.** `DraftsCount = 3` and `DeletedCount = 16` are hardcoded in `MessageStore`. They are never decremented by any UI action.

3. **"RE:" heuristic.** The "You replied to this message on…" date in `RightSpine` is calculated as `SentDate - TimeSpan(3, 3, 3)` — not a real reply timestamp. This is placeholder logic.

4. **Overflow strip icon lookup.** `LeftSpine.AddOverflowItems()` uses `stackStrip.Items[idx].Tag as string` as the resource key for `Properties.Resources.ResourceManager.GetObject(name)`. If a button has no `Tag`, it gets no overflow icon. The Tags are the resource names without extension.

5. **`FolderView` design-mode guard.** `folderTreeView.DrawMode` is set to `OwnerDrawText` only at runtime (`if (null == this.Site || !this.Site.DesignMode)`). In design mode the tree renders normally.

6. **`RightSpine` in design mode.** Hides `webBrowser1` in design mode to avoid WebBrowser initialization overhead in the designer.

7. **Cross-thread network event.** `NetworkChange.NetworkAvailabilityChanged` fires on a background thread. `MainForm` correctly marshals back via `this.Invoke(new MethodInvoker(this.UpdateStatusBar))`.

8. **`BindingList` vs. `DataGridView` sort.** `MessageList` calls `dataGridView1.Sort(column, direction)` which calls `ApplySortCore` on `SortableBindingList<T>`, which sorts the underlying `List<T>` in place. The grid position does not reset, but the visual row order changes.

9. **Transparent color for StackStrip button images.** `Color.FromArgb(238, 238, 238)` is the standard transparent color for all button bitmaps in this project. Read/Unread icons in MessageList use the same convention.

10. **Embedded resource namespace.** Resource stream names include the full namespace path: `"System.Windows.Forms.Samples.Mail.Inbox.xml"`. If the namespace or folder path changes, these streams will return `null`.

## Extension Points

To add a new mail folder with live count:
1. Add the count property to `MessageStore` with `INotifyPropertyChanged`
2. Add the `TreeNode` to `FolderView.InitializeComponent()`
3. Handle the property name in `FolderView.MessageStore_PropertyChanged`
4. Add a `DrawAnnotatedText` call in `folderTreeView_DrawNode`

To add a new navigation section (like Calendar view):
1. Add the `ToolStripButton` to `LeftSpine.stackStrip` in the designer
2. Give it a unique `Tag` (must match a bitmap resource name for the overflow icon)
3. Handle `StackStrip.CheckedChanged` in `MainForm` to switch the center/right pane content

To load real email from a different source:
- Replace the `DataSet.ReadXml` call in `MessageStore` constructor
- Keep `SortableBindingList<MailMessage>` as the collection type (required by `MessageList` binding)
- Keep `INotifyPropertyChanged` on `MailMessage`

To support actual HTML rendering from a URL instead of embedded stream:
- Replace `webBrowser1.DocumentStream = doc` in `RightSpine.Message` setter with `webBrowser1.Navigate(url)`

## Build and Run

**Requirements:** Windows, .NET Framework 4.0+, Visual Studio (any version from 2005 onward)

1. Open `OLAF.sln`
2. Build (`Ctrl+Shift+B`)
3. Run (`F5`)

No NuGet packages. No external dependencies. The app opens directly to the inbox view with sample messages loaded from the embedded `Inbox.xml`.
